Slide 7

System Architecture — معمارية النظام

Architecture & Data Flow
🧑‍💻 إيه اللي تقوله:
المشروع مبني على 3 طبقات منفصلة — الـ Decoupled Architecture:

Frontend: معمول بـ Vue.js 3 كـ SPA

Frontend (Vue.js 3): SPA — Single Page Application — بستخدم Composition API، Vue Router للتنقل بين الصفحات، و vue-i18n للغتين عربي وإنجليزي مع دعم RTL كامل. Build tool بستخدم Vite.


Backend: معمول بـ Node.js و Express.js كـ RESTful API على نمط MVC. كل request بيعدي على JWT Middleware للتحقق من الهوية، والباسوردات متشفرة باستخدام bcrypt.


Database: استخدمت MongoDB كـ NoSQL Database، وMongoose كـ ODM للتعامل مع البيانات وربط الـ Schemas بالـ Backend.


الميزة إن كل طبقة ممكن تتطور أو تتوسع بشكل مستقل.

👨‍🏫 سؤال محتمل:
"إيه هو الـ MVC؟ وكيف طبقته؟"
🧑‍💻 ردك:
M — Model: الـ Mongoose Schema — بيعرف شكل البيانات (مثلاً: Appointment Schema)
V — View: الـ Vue.js Frontend — اللي المستخدم بيشوفه
C — Controller: الـ Express Route Handler — بيستقبل الـ Request، يتعامل مع الـ Model، ويرجع الـ Response

الفايدة: كل جزء ممكن يتعدل من غير ما يأثر على التاني.
👨‍🏫 سؤال محتمل:
"ليه اخترت MongoDB بدل MySQL مثلاً؟"
🧑‍💻 ردك:
لأن البيانات الطبية مش دايماً ليها schema ثابت — السجل الطبي لكل مريض ممكن يختلف في تفاصيله. MongoDB بيديني مرونة في الشكل، أسرع في الـ development، وبيتكامل أحسن مع Node.js ecosystem. كمان أسهل في الـ horizontal scaling لو عدد المستخدمين كبر.
⚡ لو سألك: "إيه هو الـ JWT؟"
ردك: "JWT أو JSON Web Token ده Token بيتبعت مع كل request بعد تسجيل الدخول. الـ Server بيولده وبيوقعه بـ secret key. كل request جاي، الـ middleware بيتحقق من الـ Token ده — لو صح، يكمل، لو غلط أو مش موجود، يرجع 401 Unauthorized."

Technologies Used — التقنيات المستخدمة

Tech Stack
على السلايدVue.js 3 · Vue Router · vue-i18n · Node.js · Express · MongoDB · Mongoose · JWT · bcrypt · Vite · Git · Postman · AI API · VS Code
🧑‍💻 إيه اللي تقوله:
الـ Stack بتاعي مقسم لـ 5 مجموعات:
Frontend: Vue.js 3 + Vue Router + vue-i18n + Vite
Backend: Node.js 18 + Express.js 4
Database: MongoDB 6 + Mongoose 7
Auth/Security: JWT + bcrypt
Tooling: Git/GitHub + Postman + VS Code + AI Chatbot API
👨‍🏫 سؤال محتمل:
"إيه هو الـ JWT وازاي بيشتغل؟"
🧑‍💻 ردك:
JWT أو JSON Web Token ده token مشفر بيتولد بعد تسجيل الدخول. بيتكون من 3 أجزاء: Header + Payload (بيانات المستخدم) + Signature.
كل request بعد كده بيتبعت معاه الـ Token في الـ Header. الـ Middleware في الـ Backend بيتحقق منه — لو صح يكمل، لو غلط أو منتهي الصلاحية بيرجع 401 Unauthorized.
👨‍🏫 سؤال محتمل:
"إيه الفرق بين JWT وSession؟"
🧑‍💻 ردك:
الـ Session بيتخزن على الـ Server — يعني لو عندي ملايين مستخدمين، الـ Server هيحتفظ بملايين Session، ده عبء كبير. الـ JWT Stateless — الـ Server مش بيحتفظ بأي حاجة، كل المعلومات جوا الـ Token نفسه. ده بيخليه أسهل في الـ Scaling.

Data Model — نموذج البيانات

Database Design
على السلايد6 Collections: Patient · Doctor · Appointment · Medical History · Nursing Request · Security Layer
🧑‍💻 إيه اللي تقوله:
الـ Database عندي 5 Collections رئيسية في MongoDB:

patients: بيانات المريض + الباسورد متشفر بـ bcrypt
doctors: التخصص، المدينة، التقييم، أوقات الإتاحة
appointments: booking ID، مرجع للمريض، مرجع للدكتور، التاريخ، الوقت، الحالة
medical_history: التشخيص، الأدوية، الملاحظات
nursing_requests: نوع الخدمة، العنوان، الحالة
👨‍🏫 سؤال محتمل:
"إيه هو الـ Mongoose؟"
🧑‍💻 ردك:
Mongoose هو ODM — Object Document Mapper. بيسمحلي أعرف الـ Schema (شكل البيانات) بـ JavaScript objects، ويتحقق من البيانات قبل ما تتحفظ، ويحول الـ MongoDB Documents لـ JavaScript Objects بشكل تلقائي. يعني بدله كنت هكتب MongoDB queries raw وده أصعب وأبطأ في التطوير.

Methodology — المنهجية

Development Approach
على السلايدAgile · MVC · SPA · Decoupled · Git Feature-Branch
🧑‍💻 إيه اللي تقوله:
اتبعت منهجية Agile على 4 Sprints: Requirements → Design → Implementation → Testing.
الاختيار ده خلاني أقدر أعدل وأحسن بشكل مستمر بناءً على نتائج الاختبارات. بدل ما أكمل المشروع كله وأكتشف إن فيه مشكلة في الآخر.
👨‍🏫 سؤال محتمل:
"إيه الفرق بين Agile وـ Waterfall؟"
🧑‍💻 ردك:
Waterfall ده نهج خطي: تخلص كل مرحلة قبل ما تبدأ التالية — Requirements ثم Design ثم Implementation ثم Testing. مشكلته إنك لو اكتشفت مشكلة في الـ Testing، هترجع للأول. Agile بيخليك تشتغل في دوائر — كل Sprint بتعمل فيه جزء صغير كامل وبتتأكد منه قبل ما تكمل. ده أنسب لمشاريع محتاجة flexibility.

Implementation — التنفيذ

Key Features Implemented
على السلايد6 Features: Bilingual · Auth · Appointment Flow · AI Chatbot · Session Reminder · Dark Mode
🧑‍💻 إيه اللي تقوله:
نفذت 6 Features رئيسية:
1. Bilingual RTL/LTR عن طريق vue-i18n — تغيير كامل للـ UI
2. JWT + bcrypt Auth مع Role-based Access
3. Appointment Flow كامل: حجز، تعديل، إلغاء، Booking ID
4. AI Chatbot للمساعدة والأسئلة
5. Session Reminder — تذكير تلقائي عند الدخول
6. Dark/Light Mode على كل الصفحات مع WCAG AA contrast
👨‍🏫 سؤال محتمل:
"إيه هو الـ bcrypt وليه مش بتخزن الباسورد عادي؟"
🧑‍💻 ردك:
bcrypt هو hashing algorithm بطيء عن قصد — بياخد الباسورد ويحوله لـ hash مش قابل للعكس. لو الـ Database اتسرق، الهاكر هيلاقي hashes مش passwords. كمان bcrypt بيضيف "salt" — يعني حتى لو اتنين عندهم نفس الباسورد، الـ hash هيكون مختلف لكل واحد.
👨‍🏫 سؤال محتمل:
"إيه هو الـ WCAG؟"
🧑‍💻 ردك:
WCAG هو Web Content Accessibility Guidelines — معيار دولي لإمكانية الوصول للويب. الـ AA Level بيشترط نسبة تباين معينة بين النص والخلفية عشان الناس ذوي الإعاقات البصرية يقدروا يقروا. تأكدت إن التباين مظبوط في الوضع الليلي والنهاري على كل الصفحات.

Results & Evaluation — النتائج

Results & Evaluation
على السلايد11/11 أهداف · 10/10 Test Cases · ~280ms · 4 Browsers · 5 UAT · 2 Languages
🧑‍💻 إيه اللي تقوله:
النتائج كانت:
✅ 11/11 هدف اتحقق بالكامل
✅ 10/10 Test Cases نجحوا
✅ متوسط Response Time ~280ms — أقل بكتير من الحد المسموح 2000ms

👨‍🏫 سؤال محتمل:
"إيه هو الـ UAT؟"
🧑‍💻 ردك:
UAT = User Acceptance Testing. يعني بعد ما أتأكد إن الكود شغال صح (Unit + Integration Testing)، بجيب ناس حقيقيين يستخدموا النظام ويعملوا scenarios كاملة — تسجيل، حجز، إلغاء، داشبورد — وبشوف هل بيقدروا يعملوا كل حاجة من غير مشاكل أو لأ. ده بيكشف مشاكل الـ UX اللي الاختبارات التقنية مش بتكشفها.
